## 💡 איך המערכת עובדת? (הסבר קצר)

**אתה מתווך - לא מייצר את השירותים בעצמך!**

1. **ספק SMM חיצוני** (SMMPanel/SocialPanel) - מציע שירותים במחיר X
2. **אתה (SocialMax)** - קונה מהם במחיר X, מוכר ללקוחות במחיר Y (X + markup)
3. **הלקוח** - רואה את השירותים שלך, משלם לך, ואתה שולח את ההזמנה לספק

**מודל dropshipping של שירותים!**

## 🎯 הבעיה המרכזית

**הפרויקט עוצר בדיוק בנקודת החיבור לספקי שירותים חיצוניים.**

יש כל המבנה, המודלים, ה-Controllers - אבל כשמשתמש יוצר הזמנה, היא לא מתחברת לשום ספק SMM חיצוני ולכן לא מתבצעת בפועל.

---

## 📍 איפה נעצר הפרויקט?

**הנקודה הקריטית:** `backend/src/controllers/ordersController.js`

הפונקציה `createOrder()` יוצרת את ה-Order במסד הנתונים, אבל חסר החלק שב**מעביר את ההזמנה לספק SMM חיצוני**.

חסרה הפונקציה `processOrder()` שצריכה:
1. לקחת את ה-Order שנוצר
2. לזהות את הספק הנכון (לפי `service.provider.name`)
3. לשלוח בקשה ל-API של הספק
4. לשמור את `providerOrderId` מה-API
5. להמתין לעדכונים דרך Webhooks

## 🚀 הצעדים הבאים (עדיפות)

### צעד 1: יצירת Seeds (יום אחד)
**למה:** כדי שיהיו שירותים במסד הנתונים לבדיקה
```
database/seeds/
  ├── services.js  ← יצירת שירותים לדוגמה
  ├── users.js     ← יצירת משתמש אדמין
  └── run-seeds.js ← סקריפט הרצה
```

### צעד 2: בחירת ספק SMM (יום אחד)
**למה:** צריך להחליט איזה API להשתמש
- אפשרויות: SMMPanel, SocialPanel, או אחר
- לקבל API key
- לקרוא את ה-Documentation

### צעד 3: יצירת Base Service (יום אחד)
**למה:** מבנה משותף לכל האינטגרציות
```
backend/src/services/
  └── BaseService.js ← מבנה בסיסי לכל הספקים
```

### צעד 4: אינטגרציה ראשונה - Instagram (2-3 ימים)
**למה:** להוכיח שהקונספט עובד
```
backend/src/services/
  └── InstagramService.js ← חיבור ראשון לספק
```

### צעד 5: חיבור ל-OrdersController (יום אחד)
**למה:** להשלים את `createOrder()` עם `processOrder()`

### צעד 6: Webhooks (יום אחד)
**למה:** לקבל עדכונים מהספק אוטומטית
